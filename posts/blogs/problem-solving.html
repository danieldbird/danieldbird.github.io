Problem Solving

19th February 2022

How I approach a problem

When tackling a programming problem there are a lot of different methods I use to solving the problem which I will discuss, but none are more important to me personally than breaking the problem down into smaller, more achievable tasks.

Break it down

To make talking about solving a problem a little easier, lets identify a real-world problem and solve it together step-by-step.

Let's for example, pretend we are trying to solve the problem of reversing a string.

We are given this variable, and the rest is up to us.

const string = "!ereht olleH";

Ok, so first we decide whether we want this code to run immediately and be used once, or whether we want to make it a function so that if we ever need to reverse a string in the future, we have this utility available.

Making it a function seems smarter. Let's make it a function.

function reverseString() {

}

Done.

Sanity Checking

Throughout every step of the process I like to log the output, so we can compare the expected output to what is ACTUALLY being outputted.

This way, if we know we have the correct output at this stage, then anything that is unexpected following this must be an issue with our most recent change.

This is much better than writing the entire solution at once, and then trying to find where problems were introduced at the end.

Our function will need to accept a string to work on, lets do that next, and log out its contents. 

function reverseString(string) {
  console.log(string);

// "!ereht olleH"
}

Awesome, we have the correct output at this stage.

Keep it simple, stupid.

Now we need to reverse this puppy.

There are going to be numerous ways to solve any problem, but we should always start with the simplest, and move on from there. See also Occam's razor.

The data-types in javascript come with some built-in methods, does the String data-type come with a reversing function? If so, we are almost done!

I search Google for javascript reverse string

Bummer, simply looking at the results from Google, I can see there are no built-in functions to reverse a string (This is lucky, because this would be a really short story otherwise, and make our function pretty tiny.)

What I do see on the same search results page is: Array.prototype.reverse() on the Mozilla Developer Network (MDN).

If we can cut up our string into an array and reverse it, and put it back together, that might work? Lets search to see if it's possible.

I search Google for javascript cut string into array and also javascript join array.

Wow, awesome the first results are String.prototype.split() on MDN, and String.prototype.join() on MDN.

Try things

One thing at a time.

Lets chop up our string into an array of characters, and again, do a sanity check by logging it out.

We see from the documentation that the .split() method takes a string argument of where we would like to split the string. If we wanted to split by spaces we could pass in a space ' ', but in this example we need every character to have its own position in the array, spaces included. So to split it by every character, our argument will be an empty string ''. 

function reverseString(string) {
  const charsSplit = string.split('')
  console.log(charsSplit);

// ['!','e','r','e','h','t',' ',
// 'o','l','l','e','H']
}

Sweeeeeeeet! Our sanity checking log tells us it worked, so we are on the right track up to this point.

Now we can tackle the reversing! Lets read about and apply that .reverse() method we read about in the documentation, and see what happens. According to the documentation, we do not need to supply an argument to this method. 

function reverseString(string) {
  const charsSplit = string.split('')
  const charsReversed = charsSplit.reverse();
  console.log(charsReversed);

// ['H','e','l','l','o',' ',
// 't','h','e','r','e','!']
}

Ho boy, that works! And from the log we can make out the words.

We are almost done now. We just need to join the array back together with the .join() method we found, and return the output.

We see from the documentation, that just like the .split() method, we can provide it a string to separate each item of the array when joining it back together. In this case we just want it back the way it was with no alterations, so we pass it another empty string ''.

Lets try that now.

function reverseString(string) {
  const charsSplit = string.split('')
  const charsReversed = charsSplit.reverse();
  const charsJoined = charsReversed.join('');
  console.log(charsJoined);
        
// Hello there!
}

It works! now the only thing left to do to make this function ..err functional, is to return the value instead of logging it to the console! 

function reverseString(string) {
  const charsSplit = string.split('')
  const charsReversed = charsSplit.reverse();
  const charsJoined = charsReversed.join('');
  return charsJoined;
}

..Aaaaaaaannd done!

We now have a reverseString() function we can reuse any time we need to reverse a string!

Other problem solving methods

If you are ever stuck on a problem, there are other useful methods to help you.

Pseudocode

Write out the steps of the problem in plain english to give you an idea of where to start, and a list of action items to achieve on your way to solving the bigger problem.

The rubber ducky method

Explain the problem to yourself, or somebody else. As you talk through the problem you force yourself to step through it, and possibly explain it in a simpler way that might trigger a solution or next step for yourself.

Ask for help

There is no shame in asking for help.

I like to remember that everything we know is the result of being taught by somebody or something.

It is often so much more productive to ask somebody for help rather than working on the same problem for hours without progressing.

Your time could be better spent working on something else, without trying to reinvent the wheel.
